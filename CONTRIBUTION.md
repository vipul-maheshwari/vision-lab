# NLP Lab Experiments Repository - Contribution Guidelines

Welcome to the NLP Lab Experiments Repository's Contribution Guidelines! We're excited about your interest in enhancing our collection of NLP experiments. Please review these guidelines before making contributions.

## Types of Contributions

- **Adding an Experiment**: Introduce a new NLP experiment to the repository by creating a dedicated directory. Include comprehensive documentation and well-organized code to illustrate the experiment's concept effectively.

- **Improvements**: Enhance existing experiments with optimizations, bug fixes, or new functionalities.

- **Fixing Bugs**: If you've identified a bug in an experiment, you can contribute by fixing it. Use the format `git commit -m "Fixed bug - Name of Experiment"` for commit messages.

## Getting Started

1. Fork the repository to your GitHub account.

2. Create a new branch for your contribution: `git checkout -b feature/new-experiment` or `bugfix/fix-description`.

3. Implement your changes, adhering to coding standards and best practices.

4. Thoroughly test your changes.

5. Commit your changes: Use the appropriate format for commit messages based on your contribution type.

6. Push your changes to your forked repository: `git push origin feature/new-experiment`

7. Open a pull request against the `main` branch of this repository.

## Guidelines

- Maintain the directory structure and naming conventions already established in the repository.

- Provide clear and concise documentation within your experiment directory. Explain the experiment's purpose, methodology, and expected outcomes.

- Ensure well-documented code following PEP 8 style guidelines.

- When enhancing existing experiments or fixing bugs, clearly define the problem you're addressing and how your solution improves it.

## Code of Conduct

Participate in accordance with the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). All contributors are expected to adhere to these guidelines.
